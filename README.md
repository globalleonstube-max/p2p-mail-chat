Важная концепция: Как это работает?
Вместо централизованных серверов P2P-клиент использует обычную электронную почту (SMTP и IMAP) для передачи защищенных сообщений.
Каждое сообщение упаковывается в специальный MIME-формат с заголовком [P2P-MSG] или [P2P-JOIN].
IMAP IDLE позволяет клиенту получать мгновенные пуш-уведомления от почтового провайдера без постоянного опроса (polling).
Встроенный буфер агрегации (2 секунды) собирает быстро напечатанные строки в один пакет электронной почты, защищая ваш адрес от спам-фильтров и блокировок Mail-сервера.
Сценарий 1. Запуск легковесного Go-клиента в терминале (Оригинальное ядро)
Это идеальный вариант для работы без накладных расходов. Требуется только установленный компилятор Go (версии 1.20 или выше).
Шаг 1: Подготовка конфигурации (config.json)
Пользователю нужно создать текстовый файл config.json в корневой папке приложения:
code
JSON
{
  "email": "user@gmail.com",
  "password": "your-app-specific-password",
  "imapServer": "imap.gmail.com",
  "imapPort": 993,
  "smtpServer": "smtp.gmail.com",
  "smtpPort": 465,
  "bootstrapEmails": [
    "assistant@p2p.net",
    "crypto-guard@p2p.net"
  ],
  "bootstrapUrl": ""
}
⚠️ Важно: Для Gmail, Яндекс.Почты или Outlook необходимо использовать специальный «Пароль приложения» (App Password), сгенерированный в настройках безопасности вашего почтового аккаунта, а не стандартный пароль от ящика.
Шаг 2: Компиляция и запуск
Откройте терминал в папке проекта.
Скачайте необходимые зависимости Go:
code
Bash
go mod tidy
Запустите приложение:
code
Bash
go run main.go
Или скомпилируйте исполняемый бинарный файл для вашего устройства:
code
Bash

./p2p-chat
Шаг 3: Команды в консоли
После запуска утилита перейдет в интерактивный режим ожидания ввода (>):
status — проверить состояние соединения и количество найденных узлов.
nodes — отобразить список всех известных участников децентрализованной сети.
chat <email> — войти в режим реального времени со сторонним адресатом (например, chat friend@provider.com). Находясь внутри чата, вы печатаете обычные сообщения.
/exit — выйти из конкретного чата обратно в главное меню.
exit — полностью остановить демон и закрыть программу.

Если ваши пользователи предпочитают современный графический UI с живыми графиками, системными логами интерактивного консольного терминала и интеграцией с Gemini AI:
Шаг 1: Установка зависимостей среды
Убедитесь, что на устройстве установлен Node.js (v18 или новее).
Шаг 2: Настройка окружения (.env)

code
Env
GEMINI_API_KEY=your_actual_gemini_api_key

code
Bash

Bash

Шаг 4: Использование UI-панели
Режим симуляции (Network Simulation): Позволяет протестировать весь децентрализованный протокол SMTP/IMAP в виртуальном пространстве прямо сейчас, общаясь с AI-ботами assistant@p2p.net (умный помощник) или crypto-guard@p2p.net (аудитор безопасности).
Режим реального SMTP/IMAP: Введите свои параметры и почту в конфигурационный блок слева и включите тумблер real-транспорта. Все сообщения будут отправляться через реальные SMTP-сокеты и считываться при входящем трафике IMAP!
