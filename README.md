# 🌐 P2P Mail Chat — Децентрализованный чат через SMTP/IMAP

Добро пожаловать в **P2P Mail Chat**! Это децентрализованный мессенджер с открытым исходным кодом, который использует существующую глобальную инфраструктуру электронной почты (SMTP и IMAPs) в качестве транспортного уровня. 

Вся переписка синхронизируется напрямую между вашими почтовыми ящиками без участия центральных серверов мессенджера. Приложение поддерживает мгновенные уведомления с технологией **IMAP IDLE**, автоматическое фоновое рукопожатие **[P2P-JOIN]** для обнаружения узлов (peer discovery) и буферизованную группировку исходящих сообщений для предотвращения блокировок со стороны почтовых провайдеров.

В репозитории представлены два интерфейса:
1. **Go Daemon & CLI** — высокопроизводительный бэкенд на языке Go для запуска на серверах, VPS и в терминале.
2. **Sleek Web UI Controller** — веб-интерфейс на Node.js (Vite + React + Tailwind CSS) для мониторинга логов, визуального чата и простой настройки.

## 🛠 1. Настройка SMTP / IMAP для популярных почтовых провайдеров

Для работы P2P-транспорта вашему узлу необходим доступ к почтовому ящику по протоколам SMTP (для отправки) и IMAP (для получения сообщений).

> ⚠️ **ВАЖНО:** Из соображений безопасности **никогда не используйте основной пароль от вашей почты!** Все современные провайдеры требуют создания так называемого **Пароля приложения** (App Password).

Прежде всего, зарегистрируйте отдельный, чистый почтовый ящик под нужды P2P чата (например, на Gmail или Yandex).

### 📧 Настройка G-Mail (Google)
1. Перейдите в настройки вашего Google Аккаунта -> **Безопасность**.
2. Включите **Двухэтапную аутентификацию** (обязательное условие Google).
3. Перейдите в раздел **Пароли приложений** (прокрутите страницу вниз или воспользуйтесь поиском по аккаунту).
4. Выберите устройство «Другое» и введите название (например, `P2P Mail Chat`). Нажмите **Создать**.
5. Скопируйте сгенерированный 16-значный пароль (без пробелов).
6. **Параметры соединения в приложении:**
   - **SMTP-сервер:** `smtp.gmail.com`
   - **SMTP-порт:** `587` (STARTTLS) или `465` (SSL/TLS)
   - **IMAP-сервер:** `imap.gmail.com`
   - **IMAP-порт:** `993` (SSL/TLS)

---

### 📧 Настройка Yandex Mail (Яндекс.Почта)
1. Перейдите в **Яндекс ID** -> **Безопасность** -> **Почтовые программы**.
2. Установите флажок **«Разрешить доступ к почтовому ящику по протоколам IMAP и SMTP»** (если он отключен).
3. Нажмите **«Создать пароль приложения»**.
4. Выберите тип приложения «Почта», задайте имя (например, `p2p-daemon`) и подтвердите создание.
5. Скопируйте созданный пароль.
6. **Параметры соединения в приложении:**
   - **SMTP-сервер:** `smtp.yandex.ru`
   - **SMTP-порт:** `465` (SSL/TLS)
   - **IMAP-сервер:** `imap.yandex.ru`
   - **IMAP-порт:** `993` (SSL/TLS)

---

### 📧 Настройка Mail.ru
1. Перейдите в **Личный кабинет Mail.ru** -> **Безопасность** -> **Пароли для внешних приложений**.
2. Нажмите **«Добавить»**, выберите имя для ключа.
3. Введите ваш текущий пароль для подтверждения прав и скопируйте сгенерированный пароль приложения.
4. В настройках почтового ящика обязательно убедитесь, что включен доступ по IMAP.
5. **Параметры соединения в приложении:**
   - **SMTP-сервер:** `smtp.mail.ru`
   - **SMTP-порт:** `465` (SSL/TLS)
   - **IMAP-сервер:** `imap.mail.ru`
   - **IMAP-порт:** `993` (SSL/TLS)

## 💻 2. Инструкция для новичков: Первоначальный запуск на компьютере

Если вы ранее никогда не сталкивались с терминалом и компиляцией, следуйте этим простым шагам:

### Шаг 1. Установка Go
Чтобы скомпилировать исполняемый файл демона, вам понадобится компилятор языка Go:
1. Перейдите на официальный сайт [go.dev/dl](https://go.dev/dl/).
2. Скачайте установщик для вашей операционной системы (Windows `.msi`, macOS `.pkg` или Linux тарбол).
3. Запустите установщик и следуйте инструкциям на экране.
4. Чтобы проверить успешность установки, откройте командную строку (или Терминал) и введите:
   ```bash
   go version
   ```
   Если вы видите версию Go (например, `go version go1.20 ...`), значит всё получилось!

### Шаг 2. Скачивание исходного кода
Вы можете скачать архив с репозиторием и распаковать его, либо использовать Git:
```bash
git clone https://github.com/user/p2p-mail-chat.git
cd p2p-mail-chat
```

### Шаг 3. Создание конфигурационного файла
В корневом каталоге программы создайте файл `config.json` со своими параметрами почты.

**Пример структуры `config.json`:**
```json
{
  "email": "your-p2p-node@gmail.com",
  "password": "your-app-password-here",
  "imap_server": "imap.gmail.com",
  "imap_port": 993,
  "smtp_server": "smtp.gmail.com",
  "smtp_port": 587,
  "bootstrap_emails": [
    "assistant@p2p.net",
    "crypto-guard@p2p.net"
  ],
  "bootstrap_url": ""
}
```

### Шаг 4. Компиляция и запуск демона Go
Из папки с исходным кодом запустите сборку исполняемого файла:
```bash
# Эта команда автоматически скачает зависимости и соберет бинарный файл
go build -o p2p-chat main.go
```

После завершения компиляции запустите полученный файл:
* **На Linux / macOS:**
  ```bash
  ./p2p-chat config.json
  ```
* **На Windows:**
  ```cmd
  p2p-chat.exe config.json
  ```

## 🖥 3. Использование интерактивного CLI интерфейса

При запуске исполняемого файла `p2p-chat` открывается интерактивная консоль демона со встроенным эмулятором readline.

### Доступные глобальные команды:
* `status` — Показывает текущий статус демона (запущен/остановлен), адрес вашей P2P-идентификации и количество известных активных пиров в вашей локальной базе.
* `nodes` — Выводит отсортированный список всех верифицированных узлов, которые вы обнаружили или добавили в сеть.
* `chat <email>` — Вход в режим интерактивного диалога с выбранным пиром (например: `chat companion@yandex.ru`).
* `exit` — Корректно закрывает все IMAP/SMTP сокеты и завершает работу программы.

### Работа в режиме Чата:
Когда вы входите в диалог с пиром с помощью команды `chat companion@yandex.ru`, интерфейс терминала переходит в фокус беседы (меняется приветствие ввода на `companion@yandex.ru> `):
* История сообщений из локальной базы автоматически подгрузится на экран.
* Любой текст, который вы введете и отправите нажатием Enter, будет упакован и отправлен вашему собеседнику.
* Чтобы выйти из диалога обратно в главное меню терминала, введите специальную команду:
  ```bash
  /exit
  ```

## ☁️ 4. Развертывание демона на удаленном VPS (постоянный узел в сети)

Чтобы ваш узел всегда оставался онлайн, принимал сообщения мгновенно по IMAP IDLE и буферизовал исходящую очередь, рекомендуется установить его на недорогую виртуальную машину (VPS) с Linux ОС (Ubuntu/Debian) на борту.

Ниже приведены три способа обеспечения непрерывной и фоновой работы демона.

### Способ А. Запуск через фоновую утилиту Screen (самый простой метод)
Утилита `screen` позволяет запустить любой процесс в виртуальной терминальной сессии, которая не закроется при отключении от SSH.

1. Подключитесь к вашему VPS по SSH и установите screen:
   ```bash
   sudo apt-get update
   ```
   ```bash
   sudo apt-get install screen -y
   ```
2. Создайте новую виртуальную сессию:
   ```bash
   screen -S p2p-mailbox
   ```
3. Запустите вашего скомпилированного демона:
   ```bash
   ./p2p-chat config.json
   ```
4. Теперь отключитесь от сессии screen, не завершая работу программы, нажав комбинацию клавиш:
   **`Ctrl + A`**, а затем сразу клавишу **`D`** (Exit/Detach).
5. Теперь вы можете смело закрыть окно SSH клиента. Демон продолжит непрерывную работу в фоне.
6. Для возврата к консоли управления узлом, выполните подключение по SSH и напишите команду:
   ```bash
   screen -r p2p-mailbox
   ```

---

### Способ Б. Создание системной службы systemd (рекомендуемый для надежности)
Создание `systemd` демона гарантирует, что ваше приложение будет автоматически перезапускаться при случайных сбоях, а также самостоятельно запустится после любой плановой перезагрузки сервера.

1. Перенесите ваш скомпилированный файл `p2p-chat` и конфиг `config.json` в системную папку, например `/usr/local/bin/`:
   ```bash
   sudo cp p2p-chat /usr/local/bin/
   sudo cp config.json /etc/p2p-chat.json
   ```
2. Создайте файл конфигурации службы systemd:
   ```bash
   sudo nano /etc/systemd/system/p2p-chat.service
   ```
3. Вставьте следующее содержимое в файл сервиса:
   ```ini
   [Unit]
   Description=P2P Mail Chat Client Daemon
   After=network.target

   [Service]
   Type=simple
   User=root
   WorkingDirectory=/usr/local/bin
   ExecStart=/usr/local/bin/p2p-chat /etc/p2p-chat.json
   Restart=always
   RestartSec=5
   StandardOutput=syslog
   StandardError=syslog
   SyslogIdentifier=p2p-chat

   [Install]
   WantedBy=multi-user.target
   ```
   *(Сохраните файл: нажмите `Ctrl+O`, `Enter`, затем выйдете через `Ctrl+X`)*.

4. Перезагрузите диспетчер конфигурации systemd, чтобы применить изменения:
   ```bash
   sudo systemctl daemon-reload
   ```
5. Разрешите автозагрузку демона при старте сервера и запустите его прямо сейчас:
   ```bash
   sudo systemctl enable p2p-chat
   sudo systemctl start p2p-chat
   ```
6. **Полезные команды обслуживания службы:**
   * Проверить статус работы:
     ```bash
     sudo systemctl status p2p-chat
     ```
   * Чтение логов в реальном времени (полезно для отладки соединения):
     ```bash
     journalctl -u p2p-chat -f
     ```
   * Перезапуск демона (например, после обновления пароля в json):
     ```bash
     sudo systemctl restart p2p-chat
     ```

## 🔒 5. Безопасность и архитектура обмена

* **Compaction outBuffer (Буферизация):** При отправке сообщений веб-интерфейс или бэкенд упаковывает их в специальный буфер ожидания. Он склеивает все ваши сообщения к одному получателю, отправленные в пределах **2 секунд**, в одно MIME-письмо с разделителем `\n---\n`. Это защищает почтовый аккаунт от лимитов почтовых провайдеров на отправку писем в секунду.
* **IMAP IDLE:** Демон не запрашивает почту постоянными «глупыми» запросами (polling), а поддерживает открытое TCP-соединение с почтовым сервером, ожидая пуш-уведомлений о доставке. Это бережет трафик, процессор на VPS и обеспечивает доставку сообщений за доли секунды.
* **Чистка ящика:** Все входящие сообщения, содержащие заголовки протокола `[P2P-MSG]`, `[P2P-JOIN]` и `[P2P-NODES-LIST]`, автоматически считываются демоном, заносятся во внутреннюю БД и безвозвратно удаляются с почтового сервера (помечаются флагом `Deleted` и очищаются методом `Expunge`). Ваш P2P-ящик всегда остается чистым и не переполняется.

## 📱 6. Запуск на мобильных устройствах (Android / iOS)

Запуск децентрализованных P2P-узлов на смартфонах имеет специфические ограничения из-за встроенных механизмов энергосбережения операционных систем (агрессивное закрытие фоновых TCP-сокетов) и песочниц (sandbox). Тем не менее, существует **три рабочих подхода** для мобильных платформ.

### Способ 1. Remote Web UI / Progressive Web App (PWA) — Рекомендуемый
Это самый стабильный и энергоэффективный метод, который сохранит заряд батареи смартфона.
1. Разверните демона на домашнем сервере, ПК или VPS (по инструкции из **Раздела 4**).
2. Запустите Web UI Controller на вашем сервере (он слушает интерфейс и готов к работе).
3. Откройте браузер на смартфоне (Safari на iOS или Chrome на Android) и перейдите по адресу вашей веб-панели.
4. Нажмите **«Поделиться» / «Опции»** в мобильном браузере и выберите **«Добавить на экран "Домой"»** (Add to Home Screen).
5. Приложение установится как автономное **PWA**. Вы сможете запускать его в виде полноценного мобильного приложения с быстрым доступом, в то время как тяжелая фоновая работа по опросу IMAP IDLE будет выполняться сервером или VPS.

---

### Способ 2. Локальный запуск прямо на Android (через Termux)
Если вам принципиально запустить узел локально прямо на «железе» вашего Android-смартфона без внешних серверов:
1. Установите приложение **Termux** (рекомендуется скачивать актуальную сборку с [F-Droid](https://f-droid.org/packages/com.termux/), так как версия в Google Play устарела).
2. Запустите Termux и обновите репозитории:
   ```bash
   pkg update && pkg upgrade -y
   ```
3. Установите необходимые пакеты (компилятор Go, Node.js для веб-интерфейса и Git):
   ```bash
   pkg install golang nodejs git-repo -y
   ```
4. Склонируйте репозиторий проекта и перейдите в него:
   ```bash
   git clone https://github.com/user/p2p-mail-chat.git
   cd p2p-mail-chat
   ```
5. Скомпилируйте консольного демона:
   ```bash
   go build -o p2p-chat main.go
   ```
6. Настройте файл `config.json` с помощью консольного редактора (например, `nano config.json`) и запустите:
   ```bash
   ./p2p-chat config.json
   ```
*Чтобы ОС Android не «усыпляла» запущенный в фоне Termux, откройте шторку уведомлений и нажмите кнопку **"Acquire wake-lock"** в карточке Termux.*

### Способ 3. Пакетная сборка в APK/IPA (Для разработчиков)
Для дистрибуции в виде готового нативного приложения для Google Play или App Store:
1. Ядро транспортного протокола (написанное на Go) компилируется в виде мобильной разделяемой библиотеки через инструмент **`gomobile`**:
   ```bash
   # Компиляция в Android .aar / iOS Framework
   gomobile bind -target=android,ios github.com/user/p2p-mail-chat/p2pcore
   ```
2. Визуальный слой (React + Tailwind CSS) упаковывается через кроссплатформенный фреймворк **Capacitor** или **React Native**.
3. Нативный слой приложения инициализирует Go-библиотеку при запуске приложения и проксирует входящие и исходящие P2P-пакеты через нативные мосты (Bridges) в веб-интерфейс в реальном времени.
